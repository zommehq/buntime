== Caching
:order: 3

=== Estrategia de Cache

La aplicación implementa cache en dos capas:

1. **Server-side**: Cache de respuestas del GitLab API
2. **Client-side**: React Query cache (sin persistencia)

=== Cache del Servidor

[source,typescript]
----
// Cache en memoria con TTL de 15 minutos
const cache = new Map<string, { data: unknown; expires: number }>();
const CACHE_TTL = 15 * 60 * 1000; // 15 min
----

==== Beneficios

* Reduce llamadas a GitLab
* Mejora latencia (< 1ms para cache hit)
* Evita rate limiting de GitLab

==== Desventajas

* Los datos pueden estar desactualizados hasta 15 min
* Cache perdido al reiniciar
* Cada instancia tiene cache propio

=== Invalidación

==== Manual

[source,bash]
----
curl -X POST http://localhost:8000/api/cache/clear
----

==== Automática (Futuro)

Planificado para v1.2.0:

* Webhook de GitLab para invalidar cache
* Invalidación selectiva por proyecto
* TTL configurable por proyecto
