:order: 1

=== Estrutura da Interface

A interface é dividida em três áreas principais:

1. **Header**: Logo, título e links externos
2. **Sidebar**: Lista de projetos e arquivos
3. **Content**: Conteúdo do documento renderizado

=== Sidebar

A sidebar exibe:

* Lista de projetos configurados
* Expande ao clicar para mostrar arquivos
* Links rápidos para Visão Geral e Release Notes
* Arquivos .adoc do projeto

=== Navegação por Projeto

1. Clique em um projeto para expandir
2. Clique em "Visão Geral" para ver o README
3. Clique em "Release Notes" para ver o changelog
4. Clique em um arquivo para visualizar

=== Estados da Interface

* **Loading**: Spinner durante carregamento
* **Error**: Mensagem de erro com orientações
* **Empty**: Mensagem quando nenhum projeto selecionado
* **Content**: Documento renderizado

=== Responsividade

* Sidebar pode ser ocultada em telas menores
* Botão de toggle no header
* Conteúdo ocupa toda a largura quando sidebar está oculta

=== Fluxo de Navegação

[mermaid]
----
sequenceDiagram
    participant User
    participant Router
    participant Layout
    participant Hooks
    participant API
    participant DocViewer

    User->>Router: Click project link
    Router->>Layout: Navigate to /$project
    Layout->>Hooks: useProjectTree(project, lang)
    Hooks->>API: GET /api/projects/{project}/tree
    API-->>Hooks: TreeResponse
    Layout->>Layout: Render sidebar with tree

    User->>Router: Click document link
    Router->>DocViewer: Navigate to /$project/{slug}
    DocViewer->>Hooks: useProjectDoc(project, slug, lang)
    Hooks->>API: GET /api/projects/{project}/docs/{slug}
    API-->>Hooks: DocResponse with HTML
    DocViewer->>DocViewer: Process HTML (fix links)
    DocViewer->>DocViewer: Render content + TOC
----
